# ML AI 오케스트레이션 — 비즈니스 확장 방향

> 작성일: 2026-03-21
> 기반 서비스: `ai-orchestration` (Ray Serve, EKS)

---

## 핵심 AI 능력 재정의

이 서비스의 AI는 단순 딥페이크 탐지가 아니라 **"영상이 어떤 AI 툴로 만들어졌는지 식별"** 하는 시스템이다.

- 출력 예시: `"이 영상은 Sora로 만들어졌습니다 (87%)"`
- 35클래스 분류: Sora, Runway Gen2, Pika, ElevenLabs 등 + Real
- 멀티모달: 영상(시각) + 음성 + 립싱크 일치도

---

## 1. 영상 분석 UX 개선

### 변경 사항
- URL 입력 제거 → 파일 업로드만
- 업로드 전 영상 미리보기 (`<video>` 태그)
- 파일 크기/길이 제한 안내 (100MB, 3분 이하)
- 로그 터미널 → 단계별 progress bar로 교체
- 결과 화면에 공유/저장 버튼

### 비즈니스 로직 추가

#### 분석 이력 (히스토리)
- 유저가 분석한 영상 목록 + 결과 저장
- 프로필 페이지 탭으로 연동 → 재방문 유도
- 필요 작업: `user` 서비스 DB에 `analysis_history` 테이블 추가

#### 분석 횟수 제한 + 포인트 연동
| 등급 | 횟수 |
|---|---|
| 비로그인 | 1회 |
| 일반 회원 | 월 N회 |
| 프리미엄 | 무제한 |

- Redis rate limiting 활용 (이미 인프라 존재)
- 초과 시 코인으로 충전 → 기존 퀴즈/게임 포인트 시스템과 연결

## 2. 내부 서비스 확장

### 퀴즈 콘텐츠 자동 생성
- 현재: 어드민이 퀴즈 문제 수동 등록
- 개선: AI가 영상 분석 → "Sora 영상, 신뢰도 91%" → 퀴즈 문제 후보 자동 생성
- 어드민은 검수만 하면 되는 구조
- 연결 포인트: `ai-orchestration` → `admin` 서비스 → `quiz` DB

### 커뮤니티 게시글 자동 태깅
- 유저가 커뮤니티에 영상 업로드 시 AI가 자동으로 태그 부착
- 예: `#Sora`, `#Runway`, `#Real`
- 연결 포인트: `community` 서비스 업로드 훅 → `ai-orchestration` 비동기 호출

### 리포트 서비스 강화
- 현재: PDF 통계 리포트
- 개선: 개인화된 AI 탐지 능력 리포트
  - "이 유저는 Sora 영상을 잘 잡아냄, Runway는 약함"
  - 퀴즈 정답 데이터 + 분석 이력 결합
- 연결 포인트: `report` 서비스 (FastAPI) + `quiz` DB + `analysis_history`

### 퀴즈 결과 연동
- 분석 결과 JSON에 이미 `recommendation.quiz_topics` 필드 존재
- 결과 화면에 "이 분석 기반 퀴즈 풀기" 버튼 → 기존 게임화 시스템 연결

## 3. 외부 공개 API (장기)

### 개념
지금 내부 서비스용 `ai-orchestration` API를 외부에 공개해 다른 회사/개발자가 API 키로 사용할 수 있게 하는 것. OpenAI API 구조와 동일.

### 타겟
| 대상 | 니즈 |
|---|---|
| 미디어사 / 뉴스 편집국 | 제보 영상 AI 생성 여부 자동 검수 |
| 팩트체크 기관 | 선거철 대량 영상 배치 검수 |
| 중소 플랫폼 운영사 | 업로드 영상 AI 라벨 자동 부착 |

### 현재 구조 대비 추가 필요 항목
| 현재 | 추가 필요 |
|---|---|
| PawFiler 로그인 유저만 호출 가능 | API 키 인증 레이어 |
| 단건 분석 | 배치 분석 엔드포인트 |
| 결과 프론트 표시 | 웹훅 (분석 완료 시 고객사 서버로 전송) |
| 무제한 | 요금제별 호출 횟수 제한 |

### 전제 조건
- 모델 F1 ≥ 0.90 검증 완료
- 실서비스 트래픽 누적 후 진행
- **지금은 아키텍처가 B2B 전환을 막지 않는 방향으로 설계해두는 것이 목표**
  - 인증 레이어를 `user_id` 기반과 `api_key` 기반으로 분리해두면 나중에 api_key 검증 로직만 추가하면 됨
