## Observability 스택

### 파이프라인

```
파드 로그 → otel-collector DaemonSet (filelog receiver)
            → Loki (S3: pawfiler-loki-chunks, retention 3일)
            → Grafana (LogQL 쿼리)

파드 메트릭 → Prometheus (scrapeInterval: 60s)
             → AMP remote_write (namespace=pawfiler|admin만 필터링)
             → Grafana (SigV4 쿼리)

이상 감지 → AIOps (5분 주기)
           → Bedrock Claude 3.5 Haiku (us-east-1)
           → tool_use 루프 (get_pod_status, describe_pod_events,
                            get_prometheus_metrics, get_loki_logs,
                            restart_deployment)
           → SNS (pawfiler-aiops 토픽) → 알림 발송
```

### 비용 최적화 설정 (재가동 시 즉시 적용)

| 항목                | 설정                                | 효과                               |
| ------------------- | ----------------------------------- | ---------------------------------- |
| scrapeInterval      | 60s (기본 15s → 4배 감소)           | AMP 샘플 수 감소                   |
| writeRelabelConfigs | namespace=~"pawfiler\|admin"만 전송 | ~97% 감소                          |
| nodeExporter        | 비활성                              | 노드 메트릭 스크랩 제거            |
| kubeStateMetrics    | 비활성                              | 클러스터 전체 오브젝트 스크랩 제거 |
| **예상 비용**       | **~$31/월**                         | **(최적화 전 ~$3,700/월)**         |

## 재가동 체크리스트

```bash
# 1. pawfiler4-argocd 레포에서 아래 파일 수정 후 커밋/푸시

# observability/prometheus/prometheus.yaml
#   → prometheusSpec의 "replicas: 0" 줄 삭제

# observability/grafana/grafana.yaml
#   → "replicas: 0" → "replicas: 1"

# observability/loki/loki.yaml
#   → "replicas: 0" → "replicas: 1"

# observability/otel-collector/otel-collector.yaml
#   → nodeSelector 2줄(비용절감주석+observability:enabled) 삭제

# observability/aiops/deployment.yaml
#   → "replicas: 0" → "replicas: 1"

# 2. ArgoCD 자동 sync 확인
kubectl get pods -n monitoring

# 3. AIOps 동작 확인 (5분 후)
kubectl logs -n monitoring -l app=aiops --tail=30
```

**순서 주의**: Loki 먼저 Running → otel-collector 자동 연결 (Loki 다운 중 로그 유실)
