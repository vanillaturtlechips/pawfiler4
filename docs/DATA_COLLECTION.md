## 경로 2: 영상 분석 기능

사용자가 분석 목적으로 올린 영상. `video_analysis.tasks/results` 테이블에 저장됨.

신뢰도 구간별 4-트랙 분기:

| 신뢰도 | 트랙 | 처리 로직 | ML 학습 활용 |
|---|---|---|---|
| < 50% | 품질 미달 | "분석 불가" 반환, 사용자 재업로드 유도 | 제외 (노이즈 방지) |
| 50~85% | 오답노트 | Admin Service 검토 큐 전송, 관리자 레이블 확정 | 1순위 Gold Standard (가중치 1.0) |
| 85~95% | 준지도 학습 | 자동 레이블 부여 → S3 Buffer 저장 | 2순위 Pseudo-Label (가중치 0.5~0.7, 전체의 20~30% 이하, TTL 3개월) |
| ≥ 95% | 졸업 데이터 | 즉시 결과 반환, 퀴즈 생성용 활용 후 폐기/아카이브 | 제외 (오버피팅 방지, 비용 절감) |

Fine-tuning 트리거: 관리자 검수 데이터(50~85% 트랙) 1000건 누적 시

## 경로 3: 커뮤니티 업로드 (TODO)

사용자가 커뮤니티 게시글에 영상을 첨부하는 경로.

### 필요한 작업
1. `backend/proto/community.proto` — `CreatePostRequest`에 `media_url` 필드 추가
2. Community Service — S3 업로드 처리 (IRSA 권한 필요)
3. Admin 페이지 — "커뮤니티 업로드 영상 검토" 탭 추가
4. 관리자 승인 시 S3 `community-uploads/` → `ai-preprocessing/` 복사

### 흐름
```
사용자 커뮤니티 게시글 작성 + 영상 첨부
    → S3 community-uploads/ 저장
    → community.posts.media_url 저장
    → (분석 없음, 관리자 검토 대기)

관리자 Admin 페이지
    → 업로드 영상 목록 확인
    → 레이블 확정 → ai-preprocessing/ 이동 (학습 풀 편입)
    → 또는 삭제
```

### 주의사항
- 커뮤니티 업로드는 아무 영상이나 올릴 수 있어 품질 보장 불가
- 반드시 관리자 검토 후 학습 데이터 편입
- 자동 분석 트리거 없음 (비용 절감)
