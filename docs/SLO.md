# SLO (Service Level Objective) 정의

## 개요
PawFiler 서비스의 품질 기준을 정의합니다. 이 문서는 서비스의 가용성, 응답 시간, 오류율 등 핵심 성능 지표를 설정합니다.

## SLO 목표

### 1. 가용성 (Availability)
- **목표**: 99.95% (월 기준 약 22분 다운타임 허용)
- **측정 주기**: 월간
- **계산식**: (총 시간 - 다운타임) / 총 시간 × 100

### 2. 응답 시간 (Response Time)
| 지표 | 목표 | 설명 |
|------|------|------|
| 최소 | 100ms | 빠른 응답 시간 기준 |
| 최대 | 1000ms | 허용 가능한 최대 응답 시간 |
| P50 (Median) | ≤200ms | 50% 요청이 이 시간 이내 응답 |
| P95 | ≤500ms | 95% 요청이 이 시간 이내 응답 |
| P99 | ≤1000ms | 99% 요청이 이 시간 이내 응답 |

### 3. 오류율 (Error Rate)
- **목표**: 0%
- **대상 오류**: HTTP 5xx, gRPC 에러
- **측정 대상**: 모든 API 엔드포인트

### 4. 처리량 (Throughput)
- **평시**: 1,000 RPS (Requests Per Second)
- **최대 부하**: 5,000 RPS

### 5. 사용자 수
- **평시**: 1,000명 동시 접속
- **최대**: 5,000명 동시 접속

## 서비스별 SLO

### Quiz Service
- **목표 처리량**: 1,000 RPS
- **동시 사용자**: 5,000명
- **응답 시간 P95**: ≤500ms

### Community Service
- **목표 처리량**: 500 RPS
- **동시 사용자**: 2,000명
- **응답 시간 P95**: ≤500ms

## 모니터링

### 지표 수집
- **Prometheus**: 시스템 지표 및 애플리케이션 로그 수집
- **Grafana**: 대시보드 및 시각화

### 경고 기준
- 가용성 < 99.9%: 경고
- 오류율 > 0%: 긴급 경고
- P99 응답 시간 > 1000ms: 경고

## 개선 목표
- 주기적인 부하 테스트를 통해 SLO 달성 여부 확인
- 병목 지점 파악 및 최적화
- 지속적인 모니터링 및 개선
