## 2. 최적화 내역

### 2026-03-11 ~ 03-18 (1차 최적화)
- Quiz: `ORDER BY RANDOM()` → UUID 메모리 캐싱 (O(1) 랜덤 선택)
- Quiz: `GetQuestionById` O(n) 선형 탐색 → map O(1)
- Community: `GetRanking` 60초 인메모리 캐시
- Community: `DeletePost` S3 삭제 비동기 처리
- DB 커넥션 풀 조정 (과다 100 → 적정 50)

### 2026-04-02 (2차 최적화 — 이번 작업)
- Community: `GetFeed` 페이지별 인메모리 캐시 (30초 TTL)
- Community: `GetFeed` 쿼리에서 `COUNT(*) OVER()` window function 제거 → 별도 COUNT + 60초 캐시
- Community: DB 복합 인덱스 추가 `(is_admin_post DESC, created_at DESC)`
- Community: Redis pool 10 → 30, MinIdle 5, timeout 2초
- Community: userclient 프로필 gRPC 호출 2분 인메모리 캐시
- Community: UploadMedia 독립 엔드포인트 제거 (PR #191, jaewon)

| P99 응답시간 | <1000ms | - | - | 미측정 |

## 6. 1000 VUs 병목 분석 및 추가 개선 방안

### 현재 병목
- Community pod 1개로 1000 VUs 감당 한계
- GetFeed 캐시 미스 시(30초마다) DB 쿼리 지연이 P95를 끌어올림
- Quiz도 1000 VUs에서 평균 1초로 증가 — DB 커넥션 경쟁 추정

### 추가 개선 방안 (우선순위순)

| # | 방안 | 예상 효과 | 난이도 |
|---|------|----------|--------|
| 1 | **HPA 적용** — community/quiz pod 2~3개 | P95 50% 감소 | 낮음 |
| 2 | **RDS Read Replica** — 읽기 분산 | DB 부하 50% 감소 | 중 |
| 3 | **PgBouncer** — DB 커넥션 풀링 | 커넥션 경쟁 해소 | 중 |
| 4 | **GetFeed 커서 기반 페이지네이션** — OFFSET 제거 | 깊은 페이지 성능 향상 | 중 |
