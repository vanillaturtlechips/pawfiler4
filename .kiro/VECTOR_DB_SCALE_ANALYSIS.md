## 🗄️ 벡터 데이터 규모 추정

### 1. AI 모델 시그니처 (agent_core.ai_model_signatures)

```
데이터:
- 23개 AI 모델 (Sora, Runway, Pika, ...)
- 각 모델당 샘플 100개 (평균)
- 총: 2,300개 레코드

벡터 크기:
- embedding: vector(512) = 512 × 4 bytes = 2KB
- 메타데이터: ~1KB
- 총: 3KB per row

총 용량: 2,300 × 3KB = 6.9MB
```

### 2. 멀티모달 임베딩 (agent_core.multimodal_embeddings)

```
데이터:
- 영상 분석 요청: 10 requests/min × 60 min × 24 hr = 14,400/day
- 월간: 14,400 × 30 = 432,000 requests
- 1년: 432,000 × 12 = 5,184,000 requests

벡터 크기:
- video_embedding: vector(512) = 2KB
- audio_embedding: vector(768) = 3KB
- metadata: 1KB
- 총: 6KB per row

총 용량 (1년): 5,184,000 × 6KB = 31GB
```

### 3. 분석 메모리 (agent_core.analysis_memory)

```
데이터:
- 영상 분석 결과 저장
- 1년: 5,184,000 requests

벡터 크기:
- query_embedding: vector(1536) = 6KB
- analysis_result: JSONB ~2KB
- 총: 8KB per row

총 용량 (1년): 5,184,000 × 8KB = 41GB
```

### 4. 사용자 학습 프로필 (quiz.user_skill_vectors)

```
데이터:
- 1년 후 사용자: 10,000명

벡터 크기:
- skill_embedding: vector(256) = 1KB
- metadata: 0.5KB
- 총: 1.5KB per row

총 용량: 10,000 × 1.5KB = 15MB
```

### 5. 커뮤니티 게시글 임베딩 (community.post_embeddings)

```
데이터:
- 일일 게시글: 50개
- 1년: 50 × 365 = 18,250개

벡터 크기:
- content_embedding: vector(1536) = 6KB
- metadata: 1KB
- 총: 7KB per row

총 용량 (1년): 18,250 × 7KB = 128MB
```

## 💡 이것이 의미하는 것

### 1. RDS db.t3.micro로 충분한가?

**❌ 아니요**
- db.t3.micro: 1 vCPU, 1GB RAM
- 72GB 데이터 + 인덱스 → 최소 2GB RAM 필요
- 벡터 검색은 메모리 집약적
- **최소 db.t3.small (2GB RAM) 필요** → $30/월

### 2. CNPG가 필요한가?

**🤔 재평가 필요**

#### CNPG 장점 (여전히 유효)
- GitOps 통합
- HA (고가용성)
- 백업 자동화
- 비용 절감

#### 하지만 현실적으로...
```
벡터 검색 QPS: 0.5 ~ 2.5 (매우 낮음)
데이터 규모: 72GB (1년 후)
성장 속도: 느림 (교육 플랫폼 특성)
```

### 3. 최적의 선택은?

#### Option A: RDS 유지 + 업그레이드
```
현재: db.t3.micro ($15/월)
1년 후: db.t3.small ($30/월)
2년 후: db.t3.medium ($60/월)

장점:
- 관리 부담 없음
- 안정적
- 벡터 검색 부하 낮아서 충분

단점:
- 비용 증가
- GitOps 불가
```

#### Option B: CNPG 도입
```
비용: $7/월 (고정)

장점:
- 비용 절감 ($23/월 → $276/년)
- GitOps
- HA
- 학습 경험

단점:
- 초기 설정 시간
- 운영 책임
- 오버엔지니어링 가능성
```

#### Option C: 하이브리드 (추천) ⭐
```
현재 ~ 1년: RDS db.t3.micro 유지
- 벡터 DB 스키마만 준비
- 데이터 축적

1년 후: 데이터 규모 확인 후 결정
- 72GB 이상이면 CNPG 고려
- 그 이하면 RDS 유지
```
